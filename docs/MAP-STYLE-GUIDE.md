# 지도 스타일 표준 (MAP STYLE GUIDE)

모든 전투 상황도가 한 사람이 그린 것처럼 보이게 하기 위한 규범이다.
춘천–홍천·다부동이 이 표준의 기준 구현이며, `npm run validate`가 §마다 표시된
자동 검사로 준수 여부를 확인한다. 신규 전투는 `npm run battle:new <id>`로
표준 파일 세트를 생성한 뒤 저작을 시작한다.

## 파일 표준 — 모든 전투가 같은 14개 파일

```
meta.ts  days.ts  events.ts  frontlines.ts  movements.ts  units.ts
unitPositions.ts  terrain.ts  visual.ts  overview.ts  geo.ts
footnotes.ts  links.ts  index.ts          (plans.ts는 선택)
```

- **overview.ts 필수** [자동: ERROR] — 개요 패널은 `BattleOverview` 데이터만으로
  렌더된다. 컴포넌트에 전투별 텍스트를 하드코딩하지 않는다.
- **visual.ts 필수** [자동: WARN] — 지리 좌표(원본 데이터)와 지도 편집값(라벨
  문안·오프셋·노출 단계)을 분리한다. 좌표를 라벨 사정에 맞춰 옮기지 말 것.
- plans.ts(적 기도 유령 화살표 + 소인 스탬프)는 "무산된 계획" 서사가 있는
  전투만 제공한다. 없으면 범례·레이어가 자동으로 숨는다.

## §프레이밍 (meta.bbox)

- 전투 정면(주저항선·핵심 고지군)이 **화면의 과반**을 차지하게 조인다.
- 방어 대상 도시(대구·서울 등)는 **모서리에 걸치는 것**으로 충분하다 —
  중앙에 넣으려고 bbox를 넓히면 정면이 압축돼 모바일에서 뭉개진다(다부동 교훈).
- 광역 맥락(철수로·후방 축선)은 화면 밖 + 팬으로 본다. `reliefBbox`가 팬 여유를
  결정하며 bbox보다 사방 0.1° 이상 넓게 잡는다.

## §전체 보기 (선택 날짜 '전체')

전체 보기는 "전투의 구조 한 장"이다. 규칙:

- **사건 마커·라벨 ≤5** [자동: WARN >5] — 서사의 골격이 되는 사건만
  `visual.ts eventLabels`에 `showAtAll: true`로 지정한다. 나머지는 날짜 선택
  시에만 나타난다.
- **이동축 ≤3** (주공 1 + 상징 축 1 + 반격 1이 기본형) [자동: WARN >4] —
  `movementLabels`의 `showPathAtAll`로 제한한다. 축 라벨은 겹치면 과감히
  `showAtAll: false` (경로만 보여도 화살표는 읽힌다).
- **전선 날짜 라벨은 처음·마지막만** — 엔진이 자동 처리한다.
- 스탬프는 **빈 지면 또는 그것이 부정하는 기도 화살표 위**에, `scale ≤ 0.8`로.
  bbox 안에 있어야 한다 [자동: WARN].

## §라벨

- 사건 라벨 문안은 **10자 내외의 명사구** ("볼링장 야간 전차전" ○,
  "미 제27연대가 볼링장에서 반격하다" ✕).
- **지형 라벨과 명칭 중복 금지** — 사건 라벨에 산 이름이 들어가면 그 지형
  라벨은 `minZoom`으로 내린다 (예: 유학산).
- minZoom 단계: 항상 표시(앵커 도시·핵심 지형) → `1.2~1.5`(보조 지명) →
  `1.7~2.1`(세부 고지·시설). 한 화면에서 세 단계가 순차적으로 드러나야 한다.
- 라벨이 겹치면 순서대로: ① dx/dy 오프셋+leader ② anchor 반전 ③ minZoom 강등
  ④ showAtAll 해제. **좌표를 움직이는 것은 금지.**

## §지형

- **음영기복 래스터 필수** [자동: WARN] — `npm run relief:make <id>` →
  `npm run relief:webp` 두 줄로 만든다. 예외(해상 중심 작전 등)는 종이 배경 +
  도식 등고선으로 대체하되 사유를 meta 주석에 남긴다.
- 도식 등고선(`terrainContours`)은 래스터 위에서 **핵심 고지 강조용 보조
  장치**다. 전투당 3~6개, 쟁탈전이 벌어진 고지에만.
- 강은 `width`로 축척에 맞게 조절한다 (광역 전투 기준: 본류 2~2.5, 지류 1.5).
  도로 강조(`emphasis`)는 전투의 중심 축선 1개에만.
- 38선(`boundary38`)은 전투권을 지날 때만 좌표를 넣는다. 아니면 빈 배열.

## §타임라인 (days)

- 7일 이하 전투는 매일, 그 이상은 **주요 국면 7~12개 선별**.
- `headline`은 신문 제목이 아니라 상황도 주기(注記) 톤 — 명사구 나열.
- 브리핑 대본은 `activeEventIds` 순서에서 자동 생성되므로, 그 순서가
  이야기 순서가 되도록 배열한다.

## §개요 패널 (overview.ts)

- 표 머리글: 기본 '국군'/'북한'. 유엔군 참전 전투는 `rokHeader: '국군·유엔군'`.
- 회차 표기: 매일형 전투 `dayCounter: 'dplus'`(D+N), 국면형은 기본값(국면 N).
- `stats`(피해 막대)는 **사료 수치가 확정된 전투만**. 편차는 `rangeToPct`
  빗금으로 정직하게 표현하고 출처를 note에 쓴다.
- `advance`(전선 이동 차트)는 전선이 실제로 이동한 전투만 (고지전은 생략).
- 마지막 섹션 관례: 전투의 의의 → (있다면) 그늘·논쟁적 기록 → sourceNote에
  도식화 한계 고지. `plans.note`는 패널이 자동으로 덧붙인다.

## §색·기호 (엔진이 강제 — 데이터에서 재정의 금지)

- 진영색: 국군 `--rok`(청), 북한 `--nk`(적). 국면색은 목록 지도 전용.
- 사건 outcome 기호: 원(국군 우세)/마름모(북한)/반반(혼전) — 색맹 대비
  이중 부호화이므로 outcome을 정직하게 기입할 것.
- 부대 심볼은 APP-6 계열(보병 ✕, 포병 ●, 기갑 ◯…) + 제대 표시. 전투별 커스텀
  기호를 만들지 않는다.

## 달라도 되는 것

전투의 개성은 다음에서 나온다: bbox 프레이밍과 축척, 국면 구성과 headline
문체, 등고선을 줄 고지의 선택, 스탬프 문안·위치, overview의 선택 섹션
(stats/advance/그늘), plans 유무. 이 밖의 시각 요소가 "다르게" 보인다면
데이터가 아니라 표준이나 엔진을 고칠 일이다 — 전투 하나를 위해 컴포넌트에
분기를 넣지 않는다.
